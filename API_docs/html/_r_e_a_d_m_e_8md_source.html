23 Compiling Vespucci:
24 ===================
25 Compiling Vespucci from source is easy on Unix-like systems. Assuming you
26 have the proper pre-requisites (and Vespucci.pro is properly edited to reflect
27 their paths):
28 
29  qmake && make && make install
30 
31 Should do the trick. This does not require elevation as the default install path
32 is two directories above the source. You can modify PREFIX to install to a different
33 directory if you'd like. If you are not compiling with static Qt, and the Qt linked
34 libraries are not in your path, you will have to deploy Qt (using windeployqt or macdeployqt)
35 
36 Pre-requisites:
37 --------------------
38 If you can compile and install MLPACK and have the Qt framework installed,
39 then you can compile and install Vespucci. You should have the following
40 libraries on your system:
41 
42 * Qt
43 * MLPACK
44 * Armadillo (compiled shared or static library)
45 * LAPACK and BLAS (or high speed replacement like OpenBLAS, Accelerate, MKL, etc.)
46 * ARPACK (or ARPACK-ng)
47 * Boost (program_options, unit_test_framework, random, and math (c99))
48 * LibXML2
49 * Vespucci-QCP (compiled shared or static library, a fork of QCustomPlot https://github.com/dpfoose/Vespucci-QCP)
50 
51 All of the above packages, with the exception of MLPACK and QCustomPlot, are
52 readily available from most major GNU/Linux repositories. Installation on Mac
53 OS is similarly easy, though some packages may need to be compiled and installed
54 manually, or installed through MacPorts (https://www.macports.org/) or Fink
55 (http://www.finkproject.org/). I haven't included a configure script, but setting
56 directories should be easy enough.
57 
58 Because it is difficult to find Windows binaries for Armadillo, MLPACK and ARPACK,
59 and to make development on Windows easier, I have included Windows binaries of all
60 requisite libraries in the MinGW_libs branch. These are compiled with MinGW-w64
61 version 3 (GCC 4.8.2) with SEH for exception handling. A 7z archive containing the
62 Qt distribution and MinGW-w64 toolkit I use is also included in the toolkit branch.
63 
64 The included Qt profile uses the library paths used by Ubuntu's package manager.
65 Frameworks are used for Mac OS when possible.
66 
67 For Unix-like systems, shared libraries are the default. You will need to
68 compile and install static versions of these libraries if you want a stand-
69 alone executable. Linux packages, if made availible will require Qt, OpenBLAS, and
70 ARPACK (armadillo and MLPACK will be included).
